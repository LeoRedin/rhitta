# `@rhitta/mobile` API client

This directory holds the **Encore-generated TypeScript SDK** that wraps every
endpoint exposed by `apps/api`. Per [ADR-0020](../../../../../docs/adr/0020-encore-generated-api-client.md),
this is the **only** way `apps/mobile` calls the API — hand-rolled fetch wrappers
are banned (enforced by `@rhitta/biome-config/mobile-app`).

## Layout

```
api-client/
├── api-client.ts   # Generated by `encore gen client`. Committed.
├── index.ts        # Re-export hub. Import from here.
└── README.md       # This file.
```

## Regeneration

```bash
pnpm --filter @rhitta/mobile gen:api-client
```

That command runs `apps/mobile/scripts/gen-api-client.sh`, which invokes
`encore gen client rhitta --lang typescript --output …/api-client.ts`
against the local `apps/api` source.

### Prerequisites (one-time)

1. Encore CLI installed.

   The script accepts both a host install (`curl -fsSL https://encore.dev/install.sh | bash`)
   and a workspace-local install at `<repo-root>/.encore/bin/encore` (created
   the first time you run `apps/api/scripts/build-docker.sh`).

2. `encore auth login`.

   `encore gen client` reads metadata that the local Encore daemon only
   populates after the app has run at least once, and `encore run` itself
   currently requires an authenticated session to fetch secret metadata
   from `api.encore.cloud` (even though Rhitta does not store secrets
   there — Encore unconditionally tries to fetch them).

3. `pnpm --filter @rhitta/api dev` once.

   Boot the API at least once, then Ctrl-C it. This populates the Encore
   daemon's `~/Library/Caches/encore/cache/<local-id>/metadata.pb` file
   that `gen client` reads.

After that, every `pnpm --filter @rhitta/mobile gen:api-client` regenerates
the file in seconds.

## PR workflow

- The generated client is **committed** to the repo. PRs that change
  `apps/api` endpoints must include the regenerated client diff alongside
  the API change — that's the source of truth for consumers.
- CI re-runs `gen:api-client` and fails if the regenerated output differs
  from the committed file. **Currently this CI check is deferred** — see
  the `Generate API client` step in `.github/workflows/ci.yml`. The
  deferral is documented in Task 5's commit message.

## Initial state (Task 5 deferral)

The committed `api-client.ts` is a **placeholder** that throws on use.
Replacing it with the real generated client requires a developer with an
Encore Cloud login to follow the prerequisites above and commit the
output. The infrastructure (script, index, biome allowlist, future CI
hook) is wired so that "first regen replaces placeholder" is the entire
follow-up; no app code changes.
